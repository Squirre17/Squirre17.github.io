symbolic-execution-pry

Created2022-10-31|Updated2022-10-31|Static Analysis
|Post View:

# principle

全局维护两个变量，符号状态σ\sigmaσ 和符号化路径约束 PC.
通过触发 error 的路径约束，来对符号的值的求解技术就是符号执行。
和模糊测试不同的一点是，模糊测试是用大量的不明觉厉的数据对程序狠狠注入，以期触发一条 error 路径，而符号执行是对每个路径进行约束分析，求得可达到这个路径的符号的值的集合。

In fact, exactly the set of runs whose concrete values satisfy the path condition


符号状态就是一个符号到具体值的映射，执行到某一句的时候就进行这个映射关系。
到达条件语句时，就会产生另外一个实例，也就是出了两种情况，满足条件要求和不满足条件要求的实例，左边的σ\sigmaσ 不一样 PC 也不一样。

这里有两个分支，所以有三个实例 (一个实例经过一个分支变成两个实例).

# problem

在静态符号执行中（静态就是不会真的执行，而是通过读取 bin 去生成实例）

σ{ N→N0, sum→0 }\sigma \{\ N \rightarrow N_0,\ sum\rightarrow 0\ \}σ{ N→N0​, sum→0 } 的符号状态遇到 while 的判断的时候会生成

  • instance1: σ{ N→N1, sum→N0 }PC=(N0>0)\sigma \{\ N \rightarrow N_1,\ sum\rightarrow N_0\ \}\quad PC=(N_0>0)σ{ N→N1​, sum→N0​ }PC=(N0​>0)
  • instance2: σ{ N→N0, sum→0 }PC=(N0≤0)\sigma \{\ N \rightarrow N_0,\ sum\rightarrow 0\ \}\quad PC=(N_0\leq 0)σ{ N→N0​, sum→0 }PC=(N0​≤0)

对于 ins1，会继续判断 while，此时的 N 的符号为N1N_1N1​ (和原来不一样) 就会再次生成两个实例，从而无限递归。

另外对于平方求余问题没法高效求解（平方求余问题只能一个个去试，而不能直接用算术表达式求值）
比如我x=y2x = y^2x=y2 这个没求余的算术表达式在给定了 x 为 1 就能迅速求出 y=±1。

# dynamic symbolic execution

给定输入在执行的时候动态对约束反转求得另外一条分支的可行解来遍历所有路径的方法。

  • 当 x0 = 22, y1 = 7 的时候会走向分支 1，此时就是真实的运行程序了
  • 然后通过约束条件取反 构造一个新的 x2 y2 (2 ,1) 就能走到右边那个分支
    • 得到约束条件(x0=2 y0)∧(x0>y0+10)(x_0=2\ y_0)\wedge (x_0>y_0+10)(x0​=2 y0​)∧(x0​>y0​+10) 然后此时不满足第二个约束
      • 反转第二个约束 构造输入继续走到分支 3

那么对于构造输入就能用到模糊测试手段了，如果 mutator 能够自己定义的话就能实现 fuzz+se 了 (afl++)

# hybrid fuzzing

基于以上的问题，就有了 fuzz+se
用符号执行引擎去代替 mutator，作为每一个核上 fuzzer 的输入。
那么问题就在于，分配输入给每个 fuzzer 的问题，肯定是要尽可能的将不同的约束路径均匀的分配给每一个 fuzzer。
